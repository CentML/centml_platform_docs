title: 'Deploying Custom Models'
description: 'How to build your own containerized inference engines and deploy'

---

The CentML Platform simplifies deploying your custom models and AI applications. Package your model or application as a containerized HTTP server, then deploy it on CentML using the General Inference deployment option.

Here’s an example of how to build a stable diffusion model served by a FastAPI-based inference server: https://github.com/CentML/codex/tree/main/general-apps/general-inference/stable-diffusion/centml-endpoint

<Note>
The CentML Platform enforces [Pod Security Admission (PSA) baseline](https://kubernetes.io/docs/concepts/security/pod-security-standards/) controls at the namespace level. Containers can run as **any user, including root**. The following security restrictions are applied to all deployments:

- **Privilege escalation** is disabled (`allowPrivilegeEscalation: false`)
- **All Linux capabilities** are dropped (`drop: ["ALL"]`)
- **Seccomp profile** is set to `RuntimeDefault`
</Note>

<Note>
If you are building the container on MacOS, make sure to set the image platform as `linux/amd64`.
```bash
docker build --platform linux/amd64 -t stable-diffusion:latest .
```
</Note>

After building the container, push it to a container registry. The CentML Platform supports both public registries (e.g., [Docker Hub](https://hub.docker.com/)) and private registries (e.g., Amazon ECR, Google Artifact Registry, Azure Container Registry, or self-hosted registries).

For **public images**, navigate to the General Inference option in the CentML Platform and enter the image name, tag, and container port to deploy your application.

For **private images**, you will need to provide registry credentials so the platform can pull the image. You can either store credentials in your [Vault](/resources/generation_api_key) as a `Registry Credentials` item and select them during deployment, or enter the registry username and password directly when configuring the deployment. The platform automatically detects the registry from the image URL.
